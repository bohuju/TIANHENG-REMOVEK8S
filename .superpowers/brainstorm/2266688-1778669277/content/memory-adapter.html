MemoryAdapter Python 接口 & MCP 工具映射

Sherpa 工作流节点通过 MemoryAdapter 统一调用 GBrain MCP 工具

1. MemoryAdapter 核心接口

# sherpa/memory_adapter.py

class MemoryAdapter:
    """Sherpa 与 GBrain 之间的 MCP 客户端适配器"""

    def __init__(self, mcp_command: str = "gbrain", mcp_args: list = ["serve"]):
        # 启动 MCP stdio 子进程，建立 JSON-RPC 通道
        ...

    # ── 查询接口（各节点进入时调用）──

    async def query_experience(self, query: str, context: dict) -> list[MemoryHit]:
        """混合搜索：关键词 + 向量 + 图谱，返回相关历史经验"""

    async def get_page(self, slug: str) -> Page | None:
        """按 slug 精确读取一个 page"""

    async def get_target_profile(self, repo_url: str) -> Page | None:
        """读取目标仓库的累积画像（slug → fuzz/targets/<owner>-<repo>）"""

    async def find_similar_crashes(self, signature: str, limit: int = 5) -> list[Page]:
        """搜索签名相似的历史崩溃"""

    async def suggest_strategies(self, language: str, module: str) -> list[Page]:
        """查询同类语言/模块的有效策略，按 success_rate 排序"""

    # ── 写入接口（阶段结束后调用）──

    async def write_page(self, slug: str, frontmatter: dict,
                         compiled_truth: str, timeline: list[str]) -> bool:
        """写入一个 page（upsert 语义）"""

    async def add_timeline(self, slug: str, entry: str) -> bool:
        """追加一条 timeline entry"""

    async def add_link(self, from_slug: str, to_slug: str,
                       link_type: str, source: str = "sherpa") -> bool:
        """建立两个页面之间的类型化关系"""

    # ── 批量接口（非关键阶段使用）──

    async def batch_write(self, ops: list[WriteOp]) -> BatchResult:
        """攒批异步写入，失败自动重试 3 次，部分成功返回详细报告"""

    # ── 建议接口（节点进入时主动调用）──

    async def get_suggestions(self, node: str, ctx: dict) -> Suggestion | None:
        """根据节点类型和上下文，查询 GBrain 并生成结构化建议。
        返回 None 表示没有足够相关的历史经验。"""

    # ── 会话总结接口（memory-summarize 节点专用）──

    async def summarize_session(self, session: SessionData) -> str:
        """聚合整个会话数据，写入 session page + crash pages + 更新 target
        profile + 建立所有 link 关系。返回总结的 markdown。"""

2. MCP 工具映射表

MemoryAdapter 方法 调用的 GBrain MCP 工具 调用方式
query_experience() query / search hybrid search，带 intent detection
get_page() get_page 按 slug 精确读取
find_similar_crashes() search + traverse_graph 先搜索，再沿 similar_to 边扩展
suggest_strategies() query 带 tag 和 frontmatter 过滤的混合搜索
write_page() put_page upsert 语义
add_timeline() add_timeline_entry 追加结构化事件
add_link() add_link 建立类型化关系边
batch_write() put_page × N + add_link × N 顺序调用，部分失败继续，最终报告
get_suggestions() query + traverse_graph 组合调用，结果结构化封装为 Suggestion
summarize_session() 全部写入类工具 多步事务性写入

4. 容错设计

场景 策略
MCP Server 未启动 MemoryAdapter init 时自动 gbrain serve 拉起子进程
查询超时（> 5s） 返回空结果 + 记录 warning 日志，不阻塞节点
写入失败（关键阶段） 重试 3 次，仍失败则写入本地日志作为 fallback，不阻塞 fuzz
批量写入部分失败 成功的保留，失败的写入本地 recovery log，下次重试
GBrain 数据库损坏 Sherpa 正常运行不受影响，GBrain 侧 gbrain doctor 修复
