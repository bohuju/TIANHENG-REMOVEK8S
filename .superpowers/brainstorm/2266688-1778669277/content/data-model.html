GBrain Page 数据模型设计

Sherpa 在 GBrain 中存储的 5 种 page 类型、frontmatter schema 和页面间关系

1. 五种 Page 类型总览

fuzz/target-repo

目标仓库画像

记录一个被 fuzz 的仓库的累积认知：攻击面、覆盖率趋势、推荐策略、历史发现

fuzz/session

会话总结

一次完整 fuzz 会话的过程记录：各阶段结果、决策链、最终产出

fuzz/crash

崩溃分析

单个崩溃的完整分析：签名、调用栈、真伪判定、根因、修复建议

fuzz/strategy

策略经验

经过验证的 fuzz 策略：哪种 harness 模式、seed 家族、编译器选项对某类目标有效

fuzz/harness

Harness 模式

单个 harness 的结构、针对的函数、编译结果、fuzz 效果

3. 页面间关系图谱

                    ┌──────────────────────────┐
                    │  fuzz/targets/GNOME-      │
                    │  libxml2                  │
                    │  type: fuzz/target-repo   │
                    └────────┬──────┬──────────┘
                             │      │
              source ────────┘      └──────── found_in_repo
              │                                │
    ┌─────────┴──────────┐          ┌──────────┴──────────┐
    │ fuzz/sessions/      │          │ fuzz/crashes/        │
    │ libxml2-20260510    │          │ libxml2-crash-0042   │
    │ type: fuzz/session  │          │ type: fuzz/crash     │
    └──┬──────────┬───────┘          └──────────┬───────────┘
       │          │                             │
       │ generated_in ──────────────────────────┘
       │          │                             │
       │          │          similar_to ────────┤
       │          │                             │
       │    ┌─────┴──────────────┐   ┌──────────┴──────────┐
       │    │ fuzz/harnesses/     │   │ fuzz/crashes/        │
       │    │ libxml2-harness-005│   │ libxml2-crash-0031   │
       │    │ type: fuzz/harness  │   │ type: fuzz/crash     │
       │    └────────┬────────────┘   └─────────────────────┘
       │             │
       │             │ follows_pattern
       │             │
       │    ┌────────┴──────────────┐
       │    │ fuzz/strategies/       │
       │    │ c-xml-dict-asan        │
       │    │ type: fuzz/strategy    │
       └────┼────────────────────────┘
            │ applied_to (strategy → repo)
            │

  关系类型：
  source           Session 记录了哪个目标仓库
  discovered_in    Crash 在哪个会话中被发现
  found_in_repo    Crash 属于哪个目标仓库
  generated_in     Harness 在哪个会话中生成
  follows_pattern  Harness 遵循哪个策略
  applied_to       策略应用于哪个目标仓库
  similar_to       Crash 和另一个 Crash 相似

4. Page 内容结构：compiled_truth + timeline

每个 page 除了 frontmatter，还包含两部分正文：

compiled_truth — 当前最佳结论（可被新证据覆盖）

例 crash page：
"该崩溃是一个 heap-use-after-free，发生在 xmlParseElement 处理畸形实体引用时。
根因是 xmlFreeNode 释放节点后未将父节点的 children 指针置空。
该漏洞可被远程触发，严重程度高。已在 v2.13.0 中修复。"

← 这里的结论会随着后续分析更新（比如发现更多影响范围、更新修复版本）

timeline — 只追加的证据链

- 2026-05-10T14:32: fuzz session 首次触发该崩溃，ASAN 报 heap-use-after-free
- 2026-05-10T14:45: crash-triage 初步判定为疑似真实漏洞
- 2026-05-10T15:10: crash-analysis 确认 UAF，定位 xmlFreeNode 调用
- 2026-05-10T15:30: 复现成功，POC 生成
- 2026-05-11T09:00: 提交上游，分配 CVE-2026-xxxxx
- 2026-05-12T16:00: 上游确认修复于 v2.13.0
