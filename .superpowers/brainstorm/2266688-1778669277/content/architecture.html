数据流：读路径（fuzz 阶段开始前查记忆）

例：crash-triage 阶段

  1. Sherpa crash-triage 节点收到候选崩溃
  2. MemoryAdapter.query("similar crash: SIGSEGV in libxml2 parser")
  3. GBrain hybrid search 返回历史相似崩溃的 page + timeline
  4. Sherpa 对比当前崩溃特征和历史记录，辅助分诊决策
  5. 分诊完成后 → 写入新 page，用 link 关联到同一目标仓库页

数据流：写路径（阶段结束后沉淀记忆）

关键设计决策

已确定

  • GBrain 独立 Postgres，与 Sherpa DB 物理隔离
  • MemoryAdapter 作为 Sherpa 内轻量 MCP client（Python）
  • 关键阶段同步写，其余攒批异步写
  • 写入失败不阻断 fuzz 主流程（降级为本地日志）

待确认

  • GBrain MCP Server 部署方式（同 Pod sidecar / 独立服务 / stdio 子进程）
  • 查询缓存策略（MemoryAdapter 内存缓存 vs 每次都查 GBrain）
  • GBrain page 的 type 体系和 frontmatter schema
